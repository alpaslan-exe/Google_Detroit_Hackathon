# API Endpoints

Base URL: `http://localhost:8000` (default port, configurable via `PORT` env var)

---

## `GET /`

Health check.

**Response**

```json
{
  "status": "ok",
  "service": "detroit-safelease-backend"
}
```

---

## `GET /api/score`

Returns a safety score for a given latitude/longitude pair.

**Query Parameters**

| Parameter | Type  | Required | Description             |
|-----------|-------|----------|-------------------------|
| `lat`     | float | ✅        | Latitude of the address |
| `lng`     | float | ✅        | Longitude of the address |

**Example**

```bash
curl 'http://localhost:8000/api/score?lat=42.4372582&lng=-83.2453411'
```

**Response**

```json
{
  "score": 85.2,
  "label": "LOW RISK",
  "crime_count": 39,
  "crime_score": 34.1,
  "blight_count": 9,
  "blight_score": 21.0,
  "is_compliant": true,
  "compliance_score": 30
}
```

**Errors**

| Status | Condition                         |
|--------|-----------------------------------|
| `400`  | `lat` or `lng` missing or invalid |

---

## `GET /api/geocode`

Resolves a Detroit address to coordinates using the Nominatim geocoding service. Appends `, Detroit, MI` automatically if "detroit" is not already in the address string.

**Query Parameters**

| Parameter | Type   | Required | Description             |
|-----------|--------|----------|-------------------------|
| `address` | string | ✅        | Street address to geocode |

**Example**

```bash
curl 'http://localhost:8000/api/geocode?address=1234+Woodward+Ave'
```

**Response**

```json
{
  "address": "1234, Woodward Avenue, Greektown, Detroit, Wayne County, Michigan, 48201, United States",
  "lat": 42.3332378,
  "lng": -83.0479574
}
```

**Errors**

| Status | Condition                                          |
|--------|----------------------------------------------------|
| `400`  | `address` param is missing or blank                |
| `404`  | Nominatim could not resolve the address            |
| `502`  | Upstream geocoding request failed                  |

## `GET /api/score_by_address`

Full pipeline: geocodes the address, computes the safety score, and generates a plain-English explanation. This is the primary endpoint for the frontend.

**Query Parameters**

| Parameter | Type   | Required | Description             |
|-----------|--------|----------|-------------------------|
| `address` | string | ✅        | Street address to evaluate |

**Example**

```bash
curl 'http://localhost:8000/api/score_by_address?address=19935+Patton+St'
```

**Response**

```json
{
  "address": "19935, Patton Street, Evergreen Lahser 7/8, Detroit, Wayne County, Michigan, 48219, United States",
  "lat": 42.4372582,
  "lng": -83.2453411,
  "score": 85.2,
  "label": "LOW RISK",
  "crime_count": 39,
  "crime_score": 34.1,
  "blight_count": 9,
  "blight_score": 21.0,
  "is_compliant": true,
  "compliance_score": 30,
  "explanation": "A safety score of 85.2/100 suggests this area is considered low risk overall..."
}
```

**Response Fields**

| Field              | Type    | Description                                                                 |
|--------------------|---------|-----------------------------------------------------------------------------|
| `address`          | string  | Full resolved address from geocoder                                         |
| `lat`              | float   | Latitude                                                                    |
| `lng`              | float   | Longitude                                                                   |
| `score`            | float   | Overall safety score (0–100)                                                |
| `label`            | string  | `LOW RISK` (≥70), `MODERATE RISK` (45–69), or `HIGH RISK` (<45)            |
| `crime_count`      | int     | Crime incidents within 500m in the last 90 days                             |
| `crime_score`      | float   | Crime component of score (0–40)                                             |
| `blight_count`     | int     | Unresolved blight violations within 300m                                    |
| `blight_score`     | float   | Blight component of score (0–30)                                            |
| `is_compliant`     | bool    | Whether a rental registration was found within 100m (not a legal verdict)   |
| `compliance_score` | int     | Compliance component: `30` if compliant, `0` if not                         |
| `explanation`      | string  | Tenant-facing plain-English summary generated by AI (with local fallback)   |

**Errors**

| Status | Condition                                          |
|--------|----------------------------------------------------|
| `400`  | `address` param is missing or blank                |
| `404`  | Nominatim could not resolve the address            |
| `502`  | Upstream request (geocoding or ArcGIS) failed      |

---

## Error Shape

All error responses share the same shape:

```json
{
  "error": "description of what went wrong"
}
```

---

## Notes

- CORS is enabled for all endpoints.
- The `/api/score` endpoint is useful when you already have coordinates and want to skip geocoding.
- The `/api/geocode` endpoint is useful when you need coordinates before requesting a score separately.
- For most frontend use cases, `/api/score_by_address` is the only endpoint needed.
